This app will ignore the HTTP_PROXY and HTTPS_PROXY environment variables, as it does not use HTTP to connect to the database.

Both the run query and update database actions support passing in variables along with a matching format string (the only valid format descriptor will be "%s"). This can be beneficial to prevent injection attacks. As an example for the run query action, you could set the query to select * from found_ips where ipv4_val = %s;, and the format_vars to 8.8.8.8. Parameters should be a comma seperated list to match the formatted string.
